# 贡献指南

感谢你对比特币价格预测归档项目的关注！我们欢迎各种形式的贡献。

## 📊 提交新的预测数据

这是最常见的贡献方式。如果你发现了新的比特币价格预测，请按照以下步骤提交：

### 1. 准备数据

确保你的预测数据包含以下信息：
- 📅 预测发布日期
- 🏢 机构或公司名称
- 👤 预测者姓名（如果有）
- 💰 目标价格
- ⏰ 预期达到时间
- 📝 预测内容描述
- 🔗 原文链接

### 2. 数据格式

按照以下 JSON 格式准备数据：

```json
{
  "id": [下一个可用的ID],
  "date": "YYYY-MM-DD",
  "institution": "机构名称",
  "person": "姓名",
  "role": "职位",
  "targetPrice": 价格数字,
  "targetDate": "预期时间",
  "sentiment": "bullish/bearish/neutral",
  "content": "预测详细内容...",
  "sourceUrl": "https://..."
}
```

### 3. 提交方式

#### 方式一：通过 Pull Request（推荐）

1. Fork 本仓库
2. 创建新分支：`git checkout -b add-prediction-[机构名]`
3. 编辑 `data/predictions.json`，添加新数据
4. 提交更改：`git commit -m "Add prediction from [机构名]"`
5. 推送到你的 Fork：`git push origin add-prediction-[机构名]`
6. 创建 Pull Request

#### 方式二：通过 Issue

如果你不熟悉 Git，可以：
1. 创建一个新 Issue
2. 使用 "新预测数据" 标签
3. 在 Issue 中提供所有必要信息
4. 我们会帮你添加数据

## 📝 代码规范

### HTML
- 使用语义化标签
- 保持适当的缩进（2 空格）
- 添加必要的 ARIA 属性

### CSS
- 遵循现有的命名约定
- 使用 CSS 变量进行主题定义
- 保持选择器的简洁性
- 添加必要的注释

### JavaScript
- 使用现代 ES6+ 语法
- 保持函数简洁（单一职责）
- 添加适当的错误处理
- 为复杂逻辑添加注释

## 📋 数据质量标准

提交的预测数据应该：
- ✅ 来自可信的公开来源
- ✅ 包含原文链接
- ✅ 日期准确
- ✅ 价格和时间明确
- ✅ 内容描述清晰
- ❌ 不包含个人观点或编辑
- ❌ 不包含广告或推广内容

## 🔍 审核流程

1. **提交审核**：所有 PR 会在 24-48 小时内审核
2. **数据验证**：检查数据格式和内容质量
3. **测试**：确保网站功能正常
4. **合并**：审核通过后合并到主分支
5. **部署**：自动部署到生产环境
